# Business Plan Brief Design

## Use Case

用于商业计划书、内部立项汇报、融资沟通和战略项目建议。页面应像一份正式但不沉闷的商业文件：结构清楚、判断明确、图表克制、重点可扫读。

## Visual System

- 字体：Inter / PingFang SC / Microsoft YaHei。
- 主色：深墨蓝 `#10243f`，用于标题、关键框线、主按钮。
- 强调色：琥珀金 `#d99b2b`，用于机会、增长、重点标签。
- 辅助色：青绿 `#168a64`、蓝色 `#2764c5`、砖红 `#c7523a`，用于图表区分。
- 背景：暖灰白 `#f3f0ea`，纸面白 `#ffffff`。
- 圆角：8px 以下，保持正式商务感。

## Layout Rules

- 一页只解决一个商业问题，不把封面、财务、路线图硬塞到同一区域。
- 大标题要直接说明业务对象或决策主题，不使用空泛口号。
- 指标卡必须给出单位、口径或变化方向。
- 图表需要图例、坐标或明确的刻度解释。
- 折线图只用于同一时间轴上的 2-3 个趋势指标；不同单位需要指数化，例如 `首期 = 100`。
- 雷达图用于多维能力、竞争力或项目成熟度对比，不用于财务时间序列。
- 仪表盘用于单一进度、目标完成率、健康度或风险评分。

## Interaction Rules

- 可用 tab 在同一页中切换“市场 / 模式 / 财务 / 路线图”。
- 折线图支持隐藏/显示单条指标，便于演示时聚焦。
- 图表点位应带 hover tooltip。
- 仪表盘可用按钮切换不同方案或不同阶段的完成率。
- 动效只服务阅读层级：折线绘制、卡片进入、tab 切换，不做装饰性大动画。

## Required Files

- `template.html`：完整可编辑 HTML 模板。
- `template.json`：场景、语气、布局和组件元数据。
- `design.md`：视觉、图表和交互规则。
- `layouts/`：可复制的局部页面片段。
- `examples/`：至少一个填入真实业务文案的完整样例。
- `screenshots/`：模板预览截图。
