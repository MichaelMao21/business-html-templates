# Strategy Planning Brief Design

## Use Case

用于年度战略规划、BLM 战略研讨、业务设计、战略解码和管理层汇报。页面要帮助团队从“差距”出发，经过市场洞察、战略意图、创新焦点、业务设计，再落到关键任务、组织能力、人才和文化。

## Visual System

- 主色：BLM 深绿 `#00572f`，用于品牌、导航选中态和关键色块。
- BLM 亮绿：`#00bf72`，用于章节强调、框架主线、象限标注和行动路径。
- 蓝绿：`#0b6682`，用于战略矩阵、重点节点和逻辑线。
- 荧光黄绿：`#cde82a`，少量用于强调箭头或阶段提示。
- 背景：浅绿灰 `#f3f7f5`。
- 结构：差距图、市场洞察总结象限、战略意图、三年战略目标、BLM 模型、业务设计六要素、战略解码路径、能力雷达。

## BLM Framework Rules

- 战略制定必须从差距开始：区分业绩差距和机会差距。
- 市场洞察采用“五看”：看环境、看行业、看客户、看竞争、看自己。
- 市场洞察总结必须沉淀到大战略矩阵：横轴为竞争地位，纵轴为市场增长，象限包括扭转型、增长型、防御型、多种经营型战略。
- 战略意图必须呈现 Why / Where / What：使命、愿景、目标，并给出示例或提示。
- 三年战略目标建议拆成财务、客户、内部运营、学习成长四个维度。
- BLM 模型拆成战略和执行两侧：战略侧包含市场洞察、战略意图、创新焦点、业务设计；执行侧包含关键任务、依赖关系、正式组织、人才、氛围与文化。
- 领导力放在模型顶部，价值观放在底部，市场结果与差距放在模型右侧。
- 业务设计至少覆盖：客户选择、价值主张、盈利模式、活动范围、持续价值、风险管理。
- 战略解码路径需要同时体现逻辑线和情感线，建议使用“准备阶段 - 激发团队 - 激发组织 - 业务突破”。

## Interaction Rules

- 顶部切换：总览 / 差距 / BLM模型 / 洞察 / 意图 / 目标 / 业务设计 / 解码路径 / 能力。
- BLM 模型节点支持点击高亮说明。
- 解码路径使用阶段推进图，不使用普通横排卡片。
- 能力雷达用于对比现状与目标，不用于展示时间序列。
