# Weekly Report Brief Design

## Use Case

用于个人周报、团队周报、项目周报和管理层同步。它的重点不是“漂亮封面”，而是让读者在 30 秒内看懂本周结论、进度、风险、下周计划和需要谁支持。

## Visual System

- 字体：Inter / PingFang SC / Microsoft YaHei。
- 主色：深绿色 `#17322a`，用于标题和结构线。
- 强调色：行动绿 `#008a5b`，用于完成状态和正向进展。
- 辅助色：蓝色 `#2c68c5`、金色 `#d69b2b`、红色 `#c84a3f`，分别表示指标、延期和风险。
- 背景：浅绿色网格 `#eef3f1`，适合正式但不沉闷的工作汇报。
- 圆角：8px 以下，保持商务报告感。

## Layout Rules

- 第一屏必须先给本周结论，不用“本周工作如下”这类低信息标题。
- 指标必须有口径：进度、完成事项、核心指标、待协同。
- 进展事项要写状态、事实、影响和 owner。
- 风险必须可跟进，不能只写“存在风险”。
- 下周计划要能转成行动项，最好有明确结果物或截止时间。

## Required Files

- `template.html`：完整可编辑 HTML 模板。
- `template.json`：场景、语气、密度和推荐结构。
- `design.md`：视觉、排版和内容规则。
- `layouts/`：可复制的局部页面片段。
- `examples/`：至少一个完整周报示例。
- `screenshots/`：模板预览截图。
